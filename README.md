# Pull Point

## Функциональность

Приложение разделено на несколько логических блоков, реализованных с использованием паттерна BLoC.

**👤 Авторизация и Пользователи**
- Вход по номеру телефона (email) через подтверждение по коду (SMS/звонок).
- Проверка уникальности имени пользователя при регистрации.
- Локальное сохранение данных пользователя с помощью Hive для автоматического входа.

**🗺️ Карта и Точки Притяжения (Pull Points)**
- Интерактивная карта на базе `flutter_map` (OpenStreetMap).
- Отображение всех созданных точек притяжения.
- Создание новой точки (название, описание, категория, подкатегории, время работы, геолокация).
- Фильтрация точек по категориям и подкатегориям.

**🧑‍🎨 Артисты**
- Просмотр списка артистов с возможностью поиска и фильтрации по категориям.
- Создание, редактирование и удаление профиля артиста (для авторизованных пользователей).
- Проверка уникальности имени артиста.
- Добавление артистов в избранное.

**⭐ Избранное**
- Отдельный экран со списком артистов, добавленных в избранное.
- Добавление и удаление артистов из избранного.

**💰 Кошелек и Финансы**
- Создание виртуального кошелька (привязка по номеру карты).
- Просмотр баланса и истории транзакций.
- Пополнение (`Input`) и вывод (`Output`) средств.
- Перевод средств другому артисту по его имени.

**🎨 Интерфейс**
- Нижняя навигационная панель (HomeBloc) для переключения между основными разделами.
- Всплывающие панели (`sliding_up_panel`) и карусели (`carousel_slider`) для удобного отображения контента.
- QR-коды: генерация (`qr_flutter`) и сканирование (`mobile_scanner`) (предположительно, для идентификации артистов или точек).

## Технологический стек

Проект написан на **Flutter** с использованием реактивного подхода.

- **Управление состоянием:** [flutter_bloc](https://pub.dev/packages/flutter_bloc) (версия 8.0.1) — четкое разделение логики и UI.
- **Локальное хранилище:** [Hive](https://pub.dev/packages/hive) + [hive_flutter](https://pub.dev/packages/hive_flutter) — быстрая NoSQL БД для кэширования данных пользователя.
- **Работа с сетью:** [http](https://pub.dev/packages/http) — для прямых HTTP-запросов к REST API.
- **Сериализация:** Автоматическая генерация адаптеров для Hive с помощью `hive_generator` и `build_runner`.
- **Карты и геолокация:**
    - `flutter_map` + `latlong2` — отрисовка карты.
    - `location` — получение геопозиции пользователя.
- **UI-кит:**
    - `equatable` — для сравнения объектов состояний и событий BLoC.
    - `bot_toast` — для красивых уведомлений.
    - `intl` — для форматирования дат и времени.
    - `sliding_up_panel`, `carousel_slider` — сложные UI-элементы.
    - `mask_text_input_formatter`, `email_validator` — валидация форм.
- **Платформозависимые модули:** Корректная настройка разрешений для Android (в манифесте) и iOS (в `Info.plist`) для камеры и геолокации.

## Начало работы

- Установленный **Flutter SDK** (версия 3.0.0 или выше, как указано в `pubspec.lock`).

### Установка и запуск

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/Churikov0112/pull_point.git

    ```

2.  **Установите зависимости:**
    ```bash
    flutter pub get
    ```

3.  **Сгенерируйте код для Hive адаптеров:**
    ```bash
    flutter pub run build_runner build --delete-conflicting-outputs
    ```

4.  **Запустите приложение:**
    - Подключите устройство или запустите эмулятор.
    - Выполните команду:
      ```bash
      flutter run
      ```
    - Или запустите проект через интерфейс вашей IDE.

## Бэкенд и API

Приложение взаимодействует с удаленным сервером. Конфигурация бэкенда находится в файле `lib/data/http_requests/backend_config/backend_config.dart`.

- **Базовый URL:** `http://pullpoint.cloud.sdcloud.io`
- **Аутентификация:** Запросы к защищенным эндпоинтам требуют заголовок `Authorization: Bearer <JWT-токен>`, который получается после успешной верификации кода.
- **Форматы данных:** Все запросы и ответы в формате JSON.

*(Примеры эндпоинтов можно найти в папке `lib/data/http_requests/`)*
