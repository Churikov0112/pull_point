Pull Point

Мобильное приложение на Flutter для поиска и поддержки уличных артистов и мероприятий. Внутри есть карта с Pull Point‑точками, лента афиш, QR‑сканер артистов, избранное и профиль с кошельком и платежами.

Возможности
- Карта с Pull Point‑точками и быстрым поиском по геолокации
- Лента афиш с фильтрами (дата, время, метро, категории)
- QR‑сканер для перехода на страницу артиста
- Избранные артисты
- Профиль пользователя/артиста, управление артистами
- Кошелек: пополнение, вывод, перевод пожертвований

Технологии
- Flutter SDK `>=2.17.3 <3.0.0`
- State management: `flutter_bloc`
- Локальное хранение: `hive`, `hive_flutter`
- Работа с API: `http`
- Карта: `flutter_map`, `latlong2`
- QR: `mobile_scanner`, `qr_flutter`

Структура проекта
- `lib/data` — HTTP‑запросы и реализации репозиториев
- `lib/domain` — модели и контракты
- `lib/presentation` — UI, blocs, screens, UI‑kit

Архитектура
Проект построен по слоистой схеме `data → domain → presentation` с разделением ответственности.
- `presentation` отвечает за UI и состояние. Экраны подписываются на состояния Bloc/Cubit и диспатчат события. В `lib/presentation/blocs` лежит бизнес‑логика уровня приложения (авторизация, избранное, кошелек, фильтры, точки).
- `domain` содержит модели и контракты (интерфейсы репозиториев). Этот слой не зависит от Flutter и конкретных источников данных.
- `data` реализует репозитории и конкретные источники данных: HTTP‑клиенты, DTO/модели ответа, маппинг в доменные модели, локальное хранение.

Поток данных
- UI инициирует событие (например, загрузка точек или перевод средств).
- Bloc вызывает метод репозитория из `domain`.
- Реализация репозитория в `data` обращается к API или локальному кешу.
- Результат преобразуется в доменные модели и возвращается в Bloc.
- Bloc эмитит новое состояние, UI перестраивается.

Состояние и навигация
- Все ключевые фичи имеют собственные Bloc‑классы и состояния.
- Общая навигация реализована через таб‑бар на `HomePage`.
- Экран авторизации выбирается на старте в зависимости от локального состояния пользователя.

Хранение и сессия
- Данные пользователя и сессия сохраняются в `Hive` (`user_data`), что позволяет восстановить авторизацию после перезапуска.

Интеграции
- Карта: `flutter_map` с тайлами Carto.
- Геолокация: пакет `location`.
- QR: `mobile_scanner`, генерация QR — `qr_flutter`.

Конфигурация API
Базовый URL бэкенда задается в `lib/data/http_requests/backend_config/backend_config.dart`:
- `BackendConfig.baseUrl`

Если нужен другой сервер, измените `baseUrl` и пересоберите приложение.
